mousePrep Reference

These are the main functions in the mousePrep package for processing mouse-tracking data including data standardization, analysis, outlier detection and aggregation. The functions are organized into categories based on their purpose, but they can be used in any order except when specified otherwise. Each function includes parameters for customization and returns a modified dataset along with an audit log of changes made.

Depending on the file format, one of the standard R functions for reading files into R can be used such as read.table or read.csv After the data is loaded, these functions can be used to clean, preprocess, and engineer features from the raw mouse-tracking data.

Data Standardization

Column Name Standardization

  • standardize_cols() — Standardize names/types/units

Case filtering

  • rm_cases() — Drop invalid participants/screens/empty trajectories
  • rm_non_mouse_events() — Remove non-mouse movement events
  • rm_spanish() — Remove language-specific responses such as Spanish responses
  • rm_resize_events() — Detect and remove resize events. Option to remove trajectory or participant

Device handling

  • mp_assign_device_flag() — Mouse vs touch classification
  • mp_remove_touch_devices() — Remove touch devices by rule

Feature Engineering

Screen Geometry

  • screen_dimension_calculator() — Compute screen width/height from inner+scroll for non empty screen widths
    • Wrapper screen dimension calculator for uas datasets

Timing & alignment

  • initiation_time() — Compute Initiation time from timestamps of trajectories
  • response_time() — Compute Response Time
  • move_time() — Move time (= RT − initiation)

Outlier detection based on timing features

After timing features are computed, outliers can be detected and handled using the following functions.
  • flag_move_time_outliers() — Flag outliers. Option to flag based on data-driven or fixed thresholds
  • rm_outliers() — Remove outliers

Repeated visits for trajectories and questions

  • rm_multiple_trajectories() — Option to keep first, last or longest trajectory when there are multiple trajectories per screen
  • flag_multiple_visits() - Flag questions with multiple visits. Option to flag for cognitive and/or follow-up questions
    • Option to remove participants or screens with multiple visits based on flag.

Merging Other Functional Data

  • extract_clicks() — Extract click time/location
  • merge_clicks() — Merge clicks data into trajectories data

Processing via mousetrap

  • mp_processing_mt() — Use analysis functions in mousetrap package to get to the final dataset

Export & reporting

  • mp_export_dataset() — Export cleaned dataset

Examples

  • data <- standardize_colnames(dat_full)
  • screen_dims <- screen_dimension_calculator(data, uas = TRUE)
